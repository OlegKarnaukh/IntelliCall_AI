üîê Google OAuth Setup

  1. Create a new project in Google Cloud Console.
  2. Enable **OAuth Consent Screen** and add email and profile scopes.
  3. Create **OAuth 2.0 Client IDs** (Web Application).
  4. Add Authorized Redirect URI: [BASE_URL]/api/auth/google/callback.
  5. Copy Client ID and Secret to your backend .env file.

üìû Twilio Configuration

Twilio is the communication backbone that enables the platform to place outbound calls and handle voice streams.

  1. Log in to the Twilio Console.
  2. Obtain your Account SID and Auth Token from the Account Dashboard.
  3. Get a voice-capable Twilio Phone Number.
  4. In the IntelliCall AI Dashboard, navigate to Settings > API Keys.
  5. Enter your SID, Token, and Phone Number (in E.164 format, e.g., +1234567890).
  6. Crucial Step: Ensure your BASE_URL is correctly set in your backend .env file (e.g., your ngrok URL or public domain). The platform uses this to tell Twilio where to send voice and status updates.

üåç Local Testing with ngrok

Twilio needs a public URL to send events (like voice streams) to your local machine. ngrok creates a secure tunnel from the internet to your local server.

1. Install ngrok

  • Windows: Download the ZIP from ngrok.com, extract it, and run the .exe.
  • macOS (Homebrew): Run brew install ngrok/ngrok/ngrok.
  • Linux: Use curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null && echo "deb https://ngrok-agent.s3.amazonaws.com buster main" | sudo tee /etc/apt/sources.list.d/ngrok.list && sudo apt update && sudo apt install ngrok.

2. Connect your account

Signup at ngrok.com and get your Authtoken. Run this in your terminal: 

ngrok config add-authtoken YOUR_AUTHTOKEN_HERE

3. Start the tunnel

Start your backend (default port 5001) and then run:

ngrok http 5001

4. Copy the URL

Copy the Forwarding URL (e.g., https://a1b2-c3d4.ngrok-free.app) and use it as your BASE_URL in the backend .env file.

Note: If using the free version, your URL changes every time you restart ngrok. Remember to update your .env and Settings accordingly!

üéôÔ∏è ElevenLabs Configuration (TTS)

ElevenLabs provides the high-quality Text-to-Speech (TTS) engine that gives your AI agent its voice.

  1. Get your API Key from the ElevenLabs Developer Settings.
  2. In the IntelliCall AI Dashboard, go to Settings > API Keys.
  3. Paste your key into the ElevenLabs API Key field and click Save Changes.

üëÇ Deepgram Configuration (STT)

Deepgram powers the Speech-to-Text (STT) engine, allowing the AI to understand the lead's voice in real-time with ultra-low latency.

  1. Create an account and generate an API key at console.deepgram.com (under the API Keys section).
  2. In the IntelliCall AI Dashboard, go to Settings > API Keys.
  3. Paste your key into the Deepgram API Key field and click Save Changes.

üß† OpenRouter Setup (LLM)

OpenRouter is used to access advanced Large Language Models (like GPT-4 or Claude 3) that drive the conversation logic and call analysis.

  1. Generate your API Key at openrouter.ai/settings/keys.
  2. In the IntelliCall AI Dashboard, go to Settings > API Keys.
  3. Paste your key into the OpenRouter API Key field and click Save Changes.

Update all the settings in the Settings page.

üí≥ Payment Gateways Setup

IntelliCall AI v2 supports 4 major payment providers. You can enable any combination of them.

Go to Admin Panel > Gateways to configure these.

Stripe

  1. Go to Stripe Developer Dashboard.
  2. Copy your Publishable Key (pk_...) and Secret Key (sk_...).
  3. Paste them into the Stripe section in the Admin Panel.
  4. Toggle Enabled to On.

PayPal

  1. Go to PayPal Developer Dashboard.
  2. Create a new App. Copy the Client ID and Secret Key.
  3. Paste them into the PayPal section in the Admin Panel.
  4. Toggle Enabled to On.
  5. For testing, refer to the PayPal Sandbox Guide.

Razorpay

  1. Go to Razorpay Dashboard.
  2. Generate a new Key. Copy Key ID and Key Secret.
  3. Paste them into the Razorpay section in the Admin Panel.

Dodo Payments

  1. Go to Dodo Payments Dashboard.
  2. Copy your API Key.
  3. If you are using Product IDs for plans, ensure the Dodo Product ID field in your Plan config matches the product ID in Dodo.
  4. Webhook Setup (Important):
    • Go to Dodo Developer Webhooks.
    • Add a new Webhook Endpoint: [BASE_URL]/api/payments/dodopayments/webhook.
    • Local Testing: Ensure you have ngrok configured (see Twilio & ngrok section) to get a public URL for your local server.

ü§ñ Creating AI Agents

Agents are the "brains" of your calls. You can customize how they speak and behave.

  1. Navigate to the AI Agents page in the dashboard.
  2. Click + Create Agent.
  3. Name: Give your agent a descriptive name (e.g., "Sales Assistant").
  4. System Prompt: Define the agent's personality and goals (e.g., "You are a helpful sales representative for a solar company...").
  5. Opening Message: This is the first thing the agent says when the lead answers. (Optional)
  6. Voice Settings: Select a voice name (like "Rachel") or provide a custom ElevenLabs Voice ID for premium results.

üìÖ Creating Campaigns

Campaigns allow you to automate calls to a list of leads using a specific AI Agent.

  1. Ensure you have imported leads in the Leads page.
  2. Go to the Campaigns page and click + Create Campaign.
  3. Select the AI Agent you want to use.
  4. Select the Leads or specific tags you want to target.
  5. Click Start on the campaign card to begin the automated calling process.

‚ùì Troubleshooting

Common Console Errors

Error Log Resolution
‚ùå Missing API keys for Custom Voice You enabled "Use Custom Voice" on an Agent but haven't provided Deepgram or ElevenLabs keys in Settings > API Keys.
‚ùå Deepgram Error / ElevenLabs Error Verify your API keys are correct and have remaining credits. Ensure your firewall isn't blocking outbound WebSocket connections.
‚ùå LLM Error / OpenRouter Error Check your OpenRouter API key and ensures you have a positive balance in your OpenRouter account.
‚ùå Twilio Voice Webhook Error Your BASE_URL is likely unreachable by Twilio. Check your ngrok tunnel and ensure the URL in .env matches.
‚ùå Could not find Agent or Lead The Call SID was received but the associated database records are missing. Ensure you haven't deleted the agent or lead while a call was in progress.

"WebSocket connection failed"

  • Ensure BASE_URL in backend .env is correct.
  • If using ngrok, ensure the tunnel is active and the URL matches.

"Google Auth Error"

  • Verify Redirect URIs in Google Console match exactly.
  • Ensure JWT_SECRET is set in backend.

"Database connection error"

  • Verify MongoDB service is running.
  • Check if the IP is whitelisted (if using MongoDB Atlas).
